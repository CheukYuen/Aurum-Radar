# Aurum Radar：海外市场战略情报 Agent 方案

## 1. 项目是什么

Aurum Radar 是一个面向品牌型企业海外运营场景的 **市场雷达型战略情报 Agent**。

它不依赖企业内部销售、库存、CRM 等敏感数据，而是主动从公开信息来源中获取海外市场动态，包括新闻、行业报告、社交媒体、电商平台公告、政府法规信息等，并将分散、碎片化、高频变化的市场信息转化为结构化情报、战略判断和部门行动建议。

它的核心定位不是普通数据看板，而是一个模拟“企业战略参谋 / 市场雷达”的 AI Agent：

```text
主动扫描公开信息
→ 筛选关键变化
→ 提炼结构化情报
→ 判断业务影响
→ 生成每日战略简报
→ 转化为部门行动建议
```

一句话概括：

**Aurum Radar 是一个帮助企业快速理解海外市场变化，并把情报转化为行动的战略辅助 Agent。**

---

## 2. 解决什么问题

品牌型企业在海外市场运营中，经常面临几个痛点。

### 2.1 信息来源分散

海外市场信息来自多个渠道：

```text
新闻网站
社交媒体
行业报告
电商平台公告
品牌官网
政府法规网站
行业协会
第三方数据平台
```

这些信息分散在不同语言、不同平台、不同格式中。业务团队需要花大量时间查找、阅读、复制、整理，效率很低。

---

### 2.2 信息更新频率高

海外市场变化很快：

```text
竞争对手推出新品
平台规则调整
社媒话题突然升温
法规政策变化
当地消费趋势变化
国际品牌进入新市场
```

这些变化可能每天都在发生，但企业内部团队很难持续、稳定、系统地跟踪。

---

### 2.3 信息难以转化为决策

很多市场信息本身只是“新闻”或“事件”，管理层真正需要的是：

```text
这件事对我们有什么影响？
是机会、风险，还是需要继续观察？
影响哪个市场？
影响哪个部门？
下一步应该做什么？
```

传统情报整理往往停留在“信息汇总”，没有完成从信息到判断、从判断到行动的转化。

---

### 2.4 不依赖内部数据也需要快速判断

赛题场景明确限制：不接入企业内部数据。

这意味着系统不能依赖销售额、库存、会员数据等内部指标，而必须基于公开信息完成外部市场感知。

Aurum Radar 解决的正是这个问题：

**在没有企业内部数据的情况下，仍然通过公开信息构建一个可用的外部市场雷达。**

---

## 3. Agent 的核心设计思路

Aurum Radar 的设计不是简单让大模型总结新闻，而是把 Agent 拆成一条清晰的情报处理链路。

## 3.1 信息扫描层

Agent 定时从多类型公开来源获取市场动态。

来源类型包括：

```text
新闻网站：品牌并购、竞品动作、市场事件
行业报告：消费趋势、市场规模、品类变化
社交媒体：话题热度、消费者偏好、KOL 内容
电商平台公告：平台规则、类目扶持、流量政策
政府 / 法规网站：进口政策、合规要求、供应链监管
品牌官网：新品发布、门店扩张、市场活动
```

这一层的目标是让 Agent 具备“市场雷达”的基础能力：主动发现变化。

---

## 3.2 情报筛选层

不是所有信息都有价值。

Agent 需要对采集到的信息进行筛选，判断哪些内容值得进入战略分析。

筛选维度包括：

```text
是否与目标市场相关
是否与目标行业相关
是否涉及竞争、产品、渠道、社媒、法规等关键维度
是否具有业务影响
是否是高优先级变化
是否重复或低价值
```

这一层解决的问题是：从大量公开信息中筛出真正值得关注的情报。

---

## 3.3 事件结构化层

筛选后的信息会被转化为标准化事件。

每条事件不只是原文摘要，而是被拆解为结构化字段：

```text
事件标题
事件类型
影响市场
影响维度
相关品牌
来源类型
发布时间
事件摘要
潜在影响
机会 / 风险 / 需关注
可信度
来源引用
```

例如：

```text
事件类型：竞争动态
影响市场：欧洲 / 亚太高端市场
影响维度：竞争
相关品牌：LVMH / Damiani / Tiffany / Cartier
影响判断：高端珠宝竞争加剧
风险等级：中高
```

这一层让情报从“非结构化文本”变成“可比较、可检索、可用于分析”的市场事件。

---

## 3.4 市场分析层

Agent 会按国家、区域、品类和主题，对结构化事件进行归纳。

分析维度包括：

```text
竞争动态
产品趋势
渠道 / 平台变化
社交媒体声量
法规政策变化
消费者偏好变化
区域机会变化
```

输出不只是事件列表，而是形成市场级判断，例如：

```text
新加坡：机会增强
中东：风险升温
欧洲：法规变化
北美：竞争加剧
```

这一层解决的问题是：让管理层能够以统一口径对比不同市场。

---

## 3.5 战略判断层

Agent 会进一步判断每个市场变化对业务的潜在影响。

输出分为三类：

```text
机会
风险
需关注事项
```

例如：

```text
机会：
新加坡高端消费恢复，乌节路与滨海湾商圈适合加大品牌曝光。

风险：
核心商圈租金高，国际品牌竞争集中，投放转化效率需要重点监控。

需关注：
节日营销窗口、竞品活动节奏、社媒话题变化。
```

这一层体现 Agent 的战略辅助价值：不是罗列信息，而是形成业务判断。

---

## 3.6 每日战略简报层

Agent 每日自动生成战略简报。

简报面向管理层和运营团队，结构固定、可快速阅读。

建议结构：

```text
每日战略简报

1. 今日重点市场
2. 关键变化
3. 业务影响判断
4. 建议观察重点
5. 后续行动方向
6. 来源引用
```

示例：

```text
今日重点市场：
- 新加坡：机会增强
- 中东：风险升温
- 欧洲：法规变化
- 北美：竞争加剧

关键变化：
- 竞争：LVMH 收购 Damiani，强化高端珠宝版图
- 产品：培育钻价格下行，高端天然钻需求分化
- 平台：亚马逊推出珠宝新品类流量扶持计划
- 社媒：TikTok 上 Old Money 珠宝风热度上升
- 法规：欧盟强化贵金属供应链尽调要求

业务影响判断：
- 机会：新加坡高端商圈适合加大品牌曝光
- 风险：欧洲合规成本可能提升
- 需关注：中东节日营销窗口与竞品投放节奏
```

这一层是赛题要求中最核心的输出形式。

---

## 3.7 行动建议层

Agent 最后将战略判断转化为部门可执行建议。

按部门拆分：

```text
市场部
产品部
渠道部
品牌部
法务合规
```

例如市场部：

```text
核心目标：
提升新加坡核心商圈品牌曝光与高净值客群触达

行动步骤：
1. 加大乌节路与滨海湾核心商圈投放
2. 策划新加坡本地节日营销活动
3. 拓展本地 KOL / 珠宝生活方式博主合作
4. 建立竞品投放监测

关联依据：
- 地图洞察：乌节路、滨海湾门店密度高
- 情报中心：高端珠宝需求与旅游零售恢复
- 风险提示：核心商圈租金较高，需提升投放转化效率
```

这一层完成从“战略情报”到“业务执行”的闭环。

# 4. Agent 的整体架构

可以把 Aurum Radar 的 Agent 架构分为 6 层。

```text
1. 公开信息采集层
2. 信息清洗与去重层
3. 情报事件抽取层
4. 市场分析与战略判断层
5. 每日战略简报生成层
6. 行动建议生成层
```

## 4.1 公开信息采集层

负责从公开渠道获取信息。

输入：

```text
新闻
社媒
行业报告
电商公告
法规政策
品牌动态
```

输出：

```text
原始信息列表
```

---

## 4.2 信息清洗与去重层

负责去除重复、低价值、无关信息。

处理内容：

```text
去重
语言统一
时间标准化
来源类型识别
市场归属识别
主题分类
```

输出：

```text
候选情报信息
```

---

## 4.3 情报事件抽取层

负责将信息转化为结构化事件。

输出字段：

```text
event_id
title
market
category
source_type
source_url
summary
impact_dimension
related_brand
confidence
published_at
```

---

## 4.4 市场分析与战略判断层

负责进行跨市场比较和业务影响判断。

判断维度：

```text
机会强度
风险强度
变化方向
影响范围
紧急程度
可信度
```

输出：

```text
市场状态
关键变化
机会 / 风险 / 需关注
```

---

## 4.5 每日战略简报生成层

负责将结构化情报汇总为管理层可读简报。

输出：

```text
每日战略简报
重点市场
关键变化
业务影响
建议观察
来源引用
```

---

## 4.6 行动建议生成层

负责把战略判断转化为部门动作。

输出：

```text
部门行动清单
行动目标
行动步骤
建议周期
责任部门
关联依据
```

# 5. 为什么是 Agent，而不是普通看板

普通看板通常只展示数据或图表。

Aurum Radar 的核心不同在于：

```text
普通看板：展示已有数据
Aurum Radar：主动发现变化、判断影响、生成简报、提出行动
```

它具备 Agent 特征：

```text
主动性：定时扫描公开市场信息
目标导向：围绕海外市场战略判断进行筛选
工具使用：调用公开信息源、搜索、解析、分类、摘要
推理能力：判断机会、风险和需关注事项
记忆能力：保留历史情报和市场状态变化
输出能力：生成每日战略简报和部门行动清单
可追溯性：每个判断都能回到来源引用
```

因此它不是一个静态 BI 系统，而是一个“战略辅助 Agent”。

# 6. 创新点

## 6.1 不依赖企业内部数据，也能形成市场判断

传统经营分析高度依赖内部数据，例如销售额、库存、会员数据、转化率等。

本方案强调：

```text
仅基于公开信息
也能构建海外市场变化雷达
```

这非常适合赛题场景，也降低了真实企业落地门槛。

---

## 6.2 从信息整合升级为战略模拟

系统不只是把新闻放在一起，而是模拟企业战略辅助角色：

```text
这个市场发生了什么？
为什么重要？
对业务有什么影响？
下一步应该关注什么？
哪个部门应该行动？
```

这比普通情报聚合更接近管理层决策需求。

---

## 6.3 统一口径的跨市场比较

不同市场的信息原本很难比较。

Aurum Radar 将信息统一成标准维度：

```text
竞争
产品
平台
社媒
法规
机会
风险
需关注
```

这样管理层可以快速横向比较：

```text
新加坡是机会增强
中东是风险升温
欧洲是法规变化
北美是竞争加剧
```

---

## 6.4 情报可追溯

大模型生成内容容易被质疑可信度。

本方案要求每条情报和判断都保留来源：

```text
来源类型
来源名称
发布时间
原文引用
可信度
Agent 判断依据
```

这样可以让用户知道：

```text
这个判断从哪里来？
依据是什么？
可信度如何？
```

这对企业场景非常重要。

---

## 6.5 从战略判断到部门行动

很多 AI 情报工具停留在“总结报告”。

Aurum Radar 更进一步，将战略判断转化为部门行动清单：

```text
市场部应该做什么
产品部应该做什么
渠道部应该做什么
品牌部应该做什么
法务合规应该做什么
```

这让系统从“信息工具”变成“运营决策辅助工具”。

---

## 6.6 适合 MVP 的轻量实现路径

方案没有依赖复杂内部系统或大量企业数据。

MVP 可以先实现：

```text
公开信息采集
结构化情报抽取
市场判断
每日简报
行动建议
可视化看板
```

后续再逐步接入：

```text
内部销售数据
门店数据
库存数据
CRM 数据
投放数据
审批流
任务系统
```

这让项目既适合黑客松快速演示，也具备真实商业扩展空间。

# 7. MVP 方案范围

## 7.1 MVP 重点做什么

MVP 阶段重点验证完整任务链：

```text
发现市场变化
→ 提炼关键情报
→ 判断业务影响
→ 生成每日战略简报
→ 输出部门行动建议
```

保留功能：

```text
1. 公开信息来源采集
2. 情报事件结构化
3. 市场机会 / 风险判断
4. 每日战略简报生成
5. 区域地图洞察
6. 部门行动建议
7. 来源引用与可追溯
```

---

## 7.2 MVP 暂不做什么

暂不做：

```text
1. 企业内部数据接入
2. 复杂权限系统
3. 多级审批流
4. 自动任务派发
5. 全局搜索
6. 多语言深度翻译工作流
7. 复杂 BI 自定义报表
8. 实时社媒舆情大盘
```

这样可以保证黑客松阶段聚焦核心价值，而不是被工程复杂度拖垮。

# 9. 项目价值总结

Aurum Radar 的价值可以总结为三句话：

```text
第一，它解决了海外市场信息分散、更新快、难以持续跟踪的问题。

第二，它把公开信息转化为结构化市场情报、业务影响判断和每日战略简报。

第三，它进一步把战略判断转化为部门行动建议，形成从市场雷达到业务执行的闭环。
```

更适合作为路演中的一句话表达：

**Aurum Radar 让企业在没有内部数据的情况下，也能拥有一个持续运行的海外市场战略雷达 Agent。**
